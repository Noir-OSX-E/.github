## Overview

macOS Dark Mode changes the system interface and Safari respects dark mode on websites that implement the CSS prefers-colour-scheme media query. The problem is that the majority of websites have never implemented native dark mode — bright white page backgrounds persist regardless of the system appearance setting, creating uncomfortable contrast between a dark system environment and glaring browser content. <a href="#">Noir</a> solves this for Safari by automatically converting every website to a <a href="#">dark theme</a> without requiring action from the user or changes from the website.

Noir's conversion is more sophisticated than colour inversion or a dark overlay. The extension analyses each page's CSS and applies a <a href="#">smart dark transformation</a> that maps background colours to dark equivalents, adjusts text colours for appropriate contrast, and handles images, form inputs, and structural elements to maintain visual legibility. The result reads naturally — dark pages that look designed rather than processed. Sites with native dark mode are detected automatically and left untouched, so carefully designed dark themes show their own implementation rather than being <a href="#">overridden</a> by an automatic conversion.

<a href="#">Per-site customisation</a> allows specific sites to be excluded, dimmed to a custom brightness, or adjusted in contrast — useful for image-heavy photography sites where viewing against white is preferable, or sites where the conversion produces an acceptable but imperfect result. The <a href="#">activation system</a> follows macOS system appearance by default, switching with the system. A separate time-based schedule activates dark mode during specific hours regardless of the system setting. Noir runs as a native <a href="#">Safari extension</a> within Apple's sandboxed extension model, providing the privacy and performance characteristics of a first-party browser feature rather than a script-injection approach — maintaining browsing speed and full compatibility with Private Browsing and Safari's <a href="#">privacy features</a>.

## Key Features

- Automatic <a href="#">dark conversion</a> for all websites without native dark mode
- <a href="#">Smart CSS analysis</a> maintaining readability and visual page hierarchy
- Native dark mode <a href="#">detection</a> leaving properly implemented sites untouched
- Per-site <a href="#">exclusions</a> and brightness and contrast adjustment
- <a href="#">System appearance sync</a> following macOS Dark Mode automatically
- Time-based and <a href="#">sunset schedule</a> for automatic dark mode activation
- Manual <a href="#">toggle</a> for instant per-session control from toolbar
- Native <a href="#">Safari extension</a> with full privacy and performance model
- Persistent <a href="#">per-site settings</a> remembered across browsing sessions

## Additional Information

Noir's value for dark mode users is most visible in the consistency it creates. Without it, macOS Dark Mode produces a patchwork — some sites dark, others glaring — that is visually more tiring than a uniformly lit browsing session. Noir eliminates this inconsistency, making Safari as coherent as the rest of the dark mode system.

The intelligent conversion that preserves page readability is what separates Noir from simpler extensions. Correct contrast between text and background, proper handling of images and form elements, and preservation of visual page structure ensure that Noir-converted pages are comfortable to read rather than technically dark but visually broken.

Per-site customisation with persistent settings reflects the reality that one conversion approach does not work optimally for every website. Sites visited regularly deserve individual tuning — Noir's persistence means configuration is set once and applied on every subsequent visit without manual adjustment.
